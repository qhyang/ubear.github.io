二分查找在旋转数组中的应用

题见此处 Find Minimum in Rotated Sorted Array ，跟上次那个单链表环的问题的出题模式有些相似，第一题考基本的，第二题考特殊情况。单链表那题第一个先判断链表是否有环，第二个如果有环的话，找出环的入口。这个找旋转数组中的最小值，第一题考虑旋转数组中没有重复时的情况，第二个复杂了点，考虑有重复的情况。

这题在《剑指offer》66页给出了解题思路。我是参照这个思路得出来解法，这本书需要好好揣摩。

分析

最笨的办法是扫描一遍，用O(N)的时间复杂度求出最小值。但是这个旋转数组是部分有序的，即：

  • 分成两部分，两部分都各自有序
  • 实际上是把一个有序数组的前一部分搬到后面来
  • 当不存在重复元素时，旋转数组的头元素一定大于其尾元素(数组还是全部有序的情况除外)
  • 最小值在两个数组的分界处，第二个数组的开头即是最小值

基于以上特点，可以想象，取一个中间值，若其比开头大，则其一定在第一部分，则最小值在它的后面。若其比结尾小，则其一定在第二个有序部分，则最小值在它前面。这个时候我们更新相应的指针，可以缩小查找范围。

但是结束条件呢？最终的结果就是两个指针相邻，头指针指向第一部分最大的那个，尾指针指向最小的那个。它们相邻。
