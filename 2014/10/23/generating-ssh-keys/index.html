Generating SSH keys

github 提供多种访问方式，有https和ssh等。在Linux下使用git时，咱们还是喜欢用ssh方式。ssh的原理是通过使用公玥和私玥来处理加密的问题。据说是防范中间人攻击，等我有时间了再具体了解下。

首先要为自己的机器产生一对公玥和私玥。公玥要给github。大概原理图如下：

  • 其实我有个疑问，就是github到底是怎么运用ssh的原理的。但是在第四个链接中找到这样的几句话：

    Public Key认证的基本原理是基于分对称加密方式，分别在服务端对一段数据通过公钥进行加密，如果客户端能够证明其可以使用私钥对这段数据进行解密，则可以说明客户端的身份。因为服务端需要使用客户端生成的密钥对的公钥对数据首先加密，所以需要先将公钥存储到服务端的密钥库（Auhtorized Key）。还记得Github中使用git协议push代码前需要先添加SSH KEY吗？

所以我大概能认为是这么样的一回事，github用用户对应的公玥给一些字符串加密，然后发给用户机。用户机用私玥解密字符串，然后将解密后的字符串发送给github，github把收到的字符串与原来的字符串比较，从而识别用户。
